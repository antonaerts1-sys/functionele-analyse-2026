De ideale to-be anno 2026

Van document-gedreven naar artefact-gedreven. De spec is een werkend prototype plus executable tests. AI versnelt elke fase, maar smaak, domeinkennis en verificatie blijven menselijk.

Hoe het vroeger ging

Lineair, document-zwaar, harde hand-off naar dev.

Hoe het in 2026 werkt

Zes fases, parallel waar mogelijk. Per fase: wat de mens doet, wat AI doet, wat eruit komt.

Hoge AI-rol
Gemiddeld
Mens leidt, AI assisteert
1
Discovery
Begrijpen wat het bedrijf echt nodig heeft
AI: hoog
Mens doet
  • Stakeholder-gesprekken voeren
  • Politieke context lezen
  • Synthese valideren
AI doet
  • Transcribeert en clustert interviews
  • Extraheert jobs-to-be-done
  • Doet competitor- en regelgeving-research
  • Synthetiseert pijnpunten
Output
  • Opinionated synthese (geen ruwe notulen)
  • Lijst probleemstellingen geprioriteerd
  • Landschapsanalyse
Tools Granola Claude Deep Research Notion AI
2
Requirements en user stories
Van behoefte naar concrete stories met acceptance criteria
AI: hoog
Mens doet
  • Scope-knopen doorhakken
  • Prioriteiten zetten
  • Edge cases bevestigen met business
AI doet
  • Haalt eisen uit oude systemen, tickets, Slack
  • Schrijft user stories in standaard formaat
  • Detecteert tegenstrijdigheden en gaten
  • Genereert edge cases die je anders mist
Output
  • User story backlog
  • Acceptance criteria per story
  • Conflict-rapport ter beslissing
Tools Linear AI Claude Productboard
3
Domain modeling
Entiteiten, relaties, state machines
AI: gemiddeld
Mens doet
  • Domeinkeuzes valideren
  • Kiezen tussen varianten
  • Naming-conventies bepalen
AI doet
  • Stelt entiteiten en relaties voor
  • Genereert ERD en state diagrams
  • Toont 2-3 model-varianten naast elkaar
Output
  • Datamodel (ERD)
  • State machines voor kernobjecten
  • Event-storming board
Tools Claude artifacts Mermaid Eraser AI
4
Flows en prototype
Direct van story naar klikbare UI. Wireframe-fase verdwijnt.
AI: hoog
Mens doet
  • UX-richting bepalen
  • Smaak en hiërarchie cureren
  • Itereren op feedback
AI doet
  • Genereert klikbaar prototype uit user story
  • Vult seed data in
  • Maakt varianten voor A/B-keuze
Output
  • Werkend prototype op publieke URL
  • Geen statische mockups meer
  • Business test op echte schermen
Tools v0 Claude artifacts Figma Make Lovable
5
Prototype als spec naar productie
Dev bouwt productieversie. Het prototype IS de referentie.
AI: hoog
Mens doet
  • Architectuurkeuzes maken
  • Security en data-modellen reviewen
  • Code-review op intent
AI doet
  • Schrijft 60-80% van boilerplate
  • Genereert tests uit user stories
  • Refactort en stelt verbeteringen voor
Output
  • Productie-code
  • Test suite die de spec belichaamt
  • Deploy pipeline
Tools Claude Code Cursor GitHub Copilot
6
Continue validatie
Drift detecteren tussen intent en implementatie
AI: gemiddeld
Mens doet
  • Beslissen wat regressie is, wat evolutie
  • Gebruikers observeren in productie
AI doet
  • Vergelijkt PR met oorspronkelijke intent
  • Genereert regression tests
  • Signaleert semantic drift
Output
  • Drift-rapport per release
  • Live test coverage
  • Loop terug naar fase 2 indien nodig
Tools Linear AI Claude Code review PostHog AI
De grote verschuivingen

Wat fundamenteel anders is dan vroeger.

Van →
Document → Artefact
De spec is geen PDF maar een werkend prototype plus tests.
Van →
Hand-off → Continue samenwerking
Analist en dev werken parallel. Vaak dezelfde persoon.
Van →
Maanden → Dagen
Eerste werkende versie binnen een week, niet een kwartaal.
Van →
Volledigheid → Smaak
AI levert volledigheid. De mens cureert, kiest, oordeelt.
