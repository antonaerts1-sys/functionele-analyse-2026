Wie raakt wie aan, en wanneer?

In het oude model waren er harde handovers: klant praat met analist, analist schrijft document, document gaat naar dev. In 2026 lopen klant, analist en dev parallel mee, met AI als gedeelde laag onder alles. Minder handovers, meer momenten samen.

Handovers: oud versus nieuw

Hard documentenwerk verdwijnt. Wat overblijft zijn lichte, gedeelde artefacten.

Vroeger

Vier harde handovers

  • Klant → Analist
    Interview-notulen die niemand teruglezt
  • Analist → Designer
    Wireframes als statisch document
  • Analist → Developer
    FAD van 60 pagina's, verouderd bij oplevering
  • Developer → Klant
    UAT na maanden stilte, ontdekking dat verkeerd ding gebouwd is
Nu

Gedeelde artefacten

  • Klant ⇄ Analist
    Synthese-doc, samen herzien in 30 minuten
  • Analist ⇄ Developer
    Datamodel-canvas, live samen bewerkt
  • Trio ⇄ Prototype
    Klikbaar artefact als single source of truth
  • Code ⇄ Tests
    Test suite is contract, AI bewaakt drift
Nieuwe rituelen

Minder documenten, maar wel ritme. Drie vaste momenten houden iedereen aangelijnd.

Wekelijks

Prototype walkthrough

klant • analist • dev

Per release

Drift check

analist • dev • AI
AI vergelijkt nieuwe code met oorspronkelijke stories. Trio bespreekt rode vlaggen voor de release uitgaat.
